API Reference

Integrate LeadPulse into your workflow with these endpoints.

Base URL http://localhost:8000
POST /api/search

Start a new lead discovery job. Returns immediately with a job ID you can poll for progress.

Request Body

industry · string, required
location · string, required
max_companies · int, 5–100 (default 30)
max_pages · int, 1–10 (default 5)
min_score · int, 0–100 (default 40)

Response

{ "job_id": 1, "status": "pending" }
GET /api/jobs/{job_id}

Poll a running job to check progress. Shows the current pipeline stage and how many companies have been processed.

Response

{
  "id": 1,
  "status": "running",
  "current_stage": "crawling",
  "total_companies": 30,
  "processed_companies": 12
}
GET /api/leads

Retrieve collected leads with optional filtering and pagination.

Query Parameters

page · int (default 1)
min_score · int, filter by minimum score
industry · string, filter by industry
city · string, filter by city

Response

{
  "total": 25,
  "page": 1,
  "leads": [{ ... }]
}
GET /export

Download all collected leads as an Excel file. No parameters needed.

Quick Start

# Start a search
curl -X POST http://localhost:8000/api/search \
  -H "Content-Type: application/json" \
  -d '{"industry":"Fintech","location":"London"}'

# Check progress
curl http://localhost:8000/api/jobs/1

# Get leads
curl http://localhost:8000/api/leads?min_score=50
